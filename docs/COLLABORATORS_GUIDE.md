## Design Standards

### Design Principles

1. **User-Centered Design** - All decisions prioritize user needs and workflows
2. **Simplicity** - Keep interfaces clean and intuitive
3. **Consistency** - Use the established design system
4. **Accessibility** - WCAG 2.1 AA compliance for color, contrast, and readability
5. **Efficiency** - Minimize steps to complete tasks

## Collaboration Workflow

### Design Contribution Process

#### 1. **Planning Phase**
- Review upcoming features in project roadmap
- Discuss requirements with team
- Create wireframes or low-fidelity sketches

#### 2. **Design Phase**
- Work in assigned sections of the Figma file
- Use components from the design system
- Document design decisions in notes
- Create multiple design options if exploring concepts

#### 3. **Review Phase**
- Request review from Lead Designer
- Respond to feedback in design comments
- Make iterations and updates
- Tag reviewers when ready for approval

#### 4. **Approval Phase**
- Address all feedback
- Get final sign-off from Lead Designer
- Move to "Approved" section
- Document as final for handoff to developers

#### 5. **Handoff Phase**
- Prepare design specifications
- Create developer handoff file with assets
- Provide component documentation
- Support developer questions

### Working on Sections

**Each designer gets assigned sections to own:**

1. **Create a branch** (if using Figma Team)
   - Branch from main design file
   - Name: `[Your Name] - [Feature]`
   - Example: `Sarah - Patient Dashboard`

2. **Make changes in your section**
   - Don't modify others' active work areas
   - Use components from the design system
   - Leave comments for documentation

3. **Request review before merging**
   - Tag Lead Designer
   - Provide context in description
   - Link to related requirements

4. **Merge after approval**
   - Update main file
   - Archive old versions
   - Update project status

### Communication During Collaboration

- **Design Decisions:** Document in Figma comments
- **Blockers:** Tag project manager immediately
- **Questions:** Use team Slack channel
- **Major Changes:** Discuss in team meetings first

## Feedback Process

### Giving Feedback

1. **Be specific** - Reference exact screens and components
2. **Be constructive** - Offer solutions, not just criticism
3. **Be respectful** - Acknowledge designer's intent
4. **Use comments** - Add directly to designs in Figma
5. **Be timely** - Provide feedback within 48 hours

### Feedback Template

```
[Component/Screen Name]
Issue: [What needs improvement]
Why: [Impact on user experience]
Suggestion: [Recommended change]
Example: [Reference similar component if applicable]
```

### Receiving Feedback

1. **Listen openly** - Understand the perspective
2. **Ask clarifying questions** - Ensure you understand
3. **Provide context** - Explain design decisions
4. **Iterate** - Make updates based on valid feedback
5. **Thank reviewers** - Acknowledge their contribution

## Frequently Asked Questions

**Q: Who do I contact for access to the Figma file?**  
A: Contact the Project Manager with your Figma email address.

**Q: Can I modify someone else's section?**  
A: Only with their permission or in emergency situations. Coordinate through Slack.

**Q: What if I find a bug or design inconsistency?**  
A: Report it in the #clinic-mgmt Slack channel with screenshots.

**Q: How often should I check for updates?**  
A: Check daily before starting work, especially before your design session.

**Q: What's the approval process for major changes?**  
A: Discuss with Lead Designer, document rationale, get sign-off before implementation.
